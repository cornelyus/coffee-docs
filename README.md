# Specialty Coffee Docs

A documentation site about specialty coffee — covering brewing methods, water chemistry, processing, tasting, origins, and equipment. Built as a portfolio project to demonstrate Next.js, MDX content pipelines, AI-powered semantic search, and multilingual content.

## Stack

- **Next.js 16.2** — App Router, Turbopack
- **Velite 0.3** — type-safe MDX content layer
- **Tailwind CSS v4** + `@tailwindcss/typography`
- **rehype-pretty-code** + **Shiki** — syntax highlighting
- **remark-gfm** — GitHub Flavored Markdown
- **Supabase** — pgvector semantic search via Edge Functions
- **Vercel Analytics** + **GoatCounter** — privacy-friendly analytics

## Content

24 MDX documents in English and Portuguese (`content/docs/en/`, `content/docs/pt/`):

- Getting started & overview
- **Brewing** — pour-over, immersion, espresso
- Processing methods
- Water chemistry
- Tasting & sensory evaluation
- Equipment
- **Origins** — overview, Ethiopia, Colombia

## Features

- **Two languages** — English (`/en/`) and Portuguese (`/pt/`), with a locale switcher in the nav
- **AI semantic search** — Cmd+K / Ctrl+K opens a search dialog powered by Supabase pgvector and OpenAI embeddings; results are scoped to the active locale
- **SEO** — sitemap, robots.txt, canonical URLs, hreflang alternates, Open Graph and Twitter card metadata

## Development

Velite and Next.js run in parallel during development:

```bash
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

Semantic search requires a Supabase project with the `search` Edge Function deployed and a populated `documents` table. To index content:

```bash
npm run index-docs
```

## Build

Velite must run before Next.js (Turbopack doesn't support webpack plugins):

```bash
npm run build
npm start
```

## Environment variables

| Variable | Required | Description |
|----------|----------|-------------|
| `NEXT_PUBLIC_SUPABASE_URL` | Yes | Supabase project URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Yes | Supabase anon key |
| `NEXT_PUBLIC_SITE_URL` | Recommended | Production URL (e.g. `https://your-domain.com`) — used for sitemap and OG tags |

## Project structure

```
content/docs/
  en/              # English MDX source files
  pt/              # Portuguese MDX source files
.velite/           # generated by Velite (gitignored)
app/               # Next.js App Router pages and layouts
  [lang]/          # locale-scoped routes (/en, /pt)
  opengraph-image.tsx  # generated OG image
  sitemap.ts       # dynamic sitemap.xml
  robots.ts        # robots.txt
components/        # UI components
lib/               # content queries, nav helpers, site URL config
scripts/           # index-docs script for seeding Supabase
velite.config.ts   # Velite schema and pipeline config
```
